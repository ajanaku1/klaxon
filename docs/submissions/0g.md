### 2. 0G Compute Â· Sealed Inference â€” TEE-attested findings gate quorum

Every Finding broadcast across the swarm carries a TEE-signed envelope

signature **locally** before counting the finding toward quorum â€” no
provider round-trip on the gossip path.

- **Provider:** `0xa48f01287233509FD694a22Bf840225062E67836` (`chatbot`,
  Qwen 2.5 7B, dstack TEE on Aliyun)
- **TEE signing address:** `0x83df4B8EbA7c0B3B740019b8c9a77ffF77D508cF`
- **Bridge:** `og-compute/summarize-finding.ts` (TypeScript, sub-processed
  by Python). Calls broker â†’ `/chat/completions` â†’ fetches signed
  attestation envelope from `/v1/proxy/signature/<chatID>`. Returns

- **Local re-verification:** `agents/finding.py::Finding.verify_tee_attestation`

### 3. 0G Storage â€” agent manifests live behind a content hash

Manifests are committed by hash to the iNFT `tokenURI`. The actual upload
to 0G Storage is currently held off-chain in `/manifests/` because both
0G TS SDK versions we tried (`@0gfoundation/0g-ts-sdk@1.2.6` and
`@0glabs/0g-ts-sdk@0.3.3`) hit an `axios`/`open-jsonrpc-provider`
incompatibility under Node 25. Documented in `FEEDBACK.md`. The
*content commitment* â€” what the iNFT actually attests to â€” is real and
verifiable by hashing the local manifest file.

## Honest scope

- **TEE signs the LLM summary, not the raw event detection.** The detector
  is deterministic Python (oracle ratio threshold). What 0G Compute
  attests is the human-readable finding summary that the operator (and
  the on-chain `FindingAttested` event) commits. This is enough to prove
  *which model reviewed the finding*, which is the slashable claim.
- **0G Storage manifests are committed by hash but uploaded post-deadline**
  due to Node 25 / SDK incompatibility â€” see FEEDBACK.
